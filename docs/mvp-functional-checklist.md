# 简易版外贸在线商店 MVP 功能清单文档

> 项目定位：面向欧美用户的 B2C 精品跨境在线商店（英文为主、中文辅助），主营玩偶 / 毛绒玩具 / 精品首饰等轻小件商品。
>
> 第一阶段目标：先完成“精品商品展示 + 购物车 + 无支付下单 + 后台导入与订单处理”的最小可用版本，验证选品、页面质量与订单转化路径。

---

## 1. 项目目标

### 1.1 商业目标
- 搭建一个视觉质量较高的双语在线商店。
- 支持少量精选商品展示与上架。
- 支持访客浏览、搜索、加入购物车、提交订单。
- 支持后台导入商品、编辑商品、处理订单、录入发货信息。
- 为后续接入支付（Stripe / PayPal）预留结构。

### 1.2 第一阶段不做
- 暂不接入在线支付。
- 暂不做复杂会员体系。
- 暂不做评论、积分、优惠券、分销、多仓、ERP/WMS。
- 暂不做自动化物流计费。
- 暂不做复杂推荐系统。

---

## 2. 目标用户与业务约束

### 2.1 目标用户
- 主要面向欧美普通消费者（B2C）。
- 默认英文界面，支持中文辅助浏览。

### 2.2 商品特点
- 商品量不大，采用“人工精选上架”。
- 商品来源于第三方平台采集，再导入后台清洗与发布。
- 商品品类包含：玩偶、毛绒玩具、精品首饰、小礼品。

### 2.3 履约方式
- 工厂代发。
- 第一阶段后台人工录入发货信息。

---

## 3. MVP 范围总览

MVP 必须覆盖以下闭环：

1. 用户可以访问高质量前台页面。
2. 用户可以搜索/浏览/查看商品。
3. 用户可以加入购物车并提交订单。
4. 后台可以管理商品与订单。
5. 管理员可以处理订单并录入物流单号。
6. 系统可以发送基础邮件通知。

## 11. 第一阶段代码目标

本阶段优先完成以下内容：

1. 初始化 Next.js 全栈项目。
2. 建立双语路由与设计系统。
3. 实现高质量前台页面骨架。
4. 建立 Prisma 数据模型。
5. 实现商品列表/详情/购物车/下单基础流程。
6. 搭建后台管理骨架页面。

---

## 12. 验收标准

满足以下条件即可认为 MVP 第一阶段完成：

- 能访问中英文首页与主要页面。
- 能浏览商品列表与商品详情。
- 能加入购物车并进入提交订单页。
- 能提交一笔无支付订单。
- 后台能查看商品与订单基础信息。
- 页面具备统一、高质量 UI 风格。
- 项目代码结构可支持后续继续扩展支付与导入功能。
